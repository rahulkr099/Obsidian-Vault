# Todo API Contract

## Base URL

```
/api/v1
```

All endpoints start with this prefix.

Example:

```
/api/v1/todos
```

---

# 1️⃣ Create Todo

## Endpoint

```
POST /todos
```

## Description

Create a new todo.

## Authentication

Required (JWT)

Header:

```
Authorization: Bearer <token>
```

---

## Request Body

```json
{
  "title": "Learn Express",
  "description": "Build REST APIs",
  "tags": ["study"]
}
```

---

## Validation

|Field|Type|Required|Description|
|---|---|---|---|
|title|String|Yes|Title of todo|
|description|String|No|Details about todo|
|tags|Array[String]|No|Tags for filtering|

---

## Success Response

Status Code

```
201 Created
```

Body

```json
{
  "success": true,
  "data": {
    "_id": "todo123",
    "title": "Learn Express",
    "description": "Build REST APIs",
    "status": "pending",
    "tags": ["study"],
    "user": "user123",
    "createdAt": "2026-03-15T10:00:00Z"
  }
}
```

---

# 2️⃣ Get All Todos

## Endpoint

```
GET /todos
```

## Description

Returns list of todos for the logged-in user.

---

## Query Parameters

|Param|Type|Description|
|---|---|---|
|page|Number|Page number|
|limit|Number|Items per page|
|status|String|Filter by status|
|tag|String|Filter by tag|
|sort|String|Sort field|
|q|String|Search text|

---

## Example Request

```
GET /todos?page=1&limit=10&status=pending&sort=-createdAt
```

---

## Success Response

```
200 OK
```

```json
{
  "success": true,
  "page": 1,
  "limit": 10,
  "results": 2,
  "data": [
    {
      "_id": "todo1",
      "title": "Learn Node",
      "status": "pending",
      "tags": ["study"]
    },
    {
      "_id": "todo2",
      "title": "Practice DSA",
      "status": "completed",
      "tags": ["coding"]
    }
  ]
}
```

# 7️⃣ Standard Error Format

All errors should follow the same structure.

```json
{
  "success": false,
  "message": "Validation failed",
  "errors": [
    {
      "field": "title",
      "message": "Title is required"
    }
  ]
}
```

---

# 8️⃣ Authentication Flow

Protected routes require JWT.

Example header:

```
Authorization: Bearer <access_token>
```

Request flow:

```
Client
   ↓
Auth Middleware
   ↓
Controller
   ↓
Database
   ↓
Response
```

---

# 9️⃣ API Versioning

Prefix all APIs with version.

```
/api/v1/todos
```

Future version:

```
/api/v2/todos
```

This prevents breaking frontend apps.
