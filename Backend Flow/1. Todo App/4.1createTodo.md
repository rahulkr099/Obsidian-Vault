# Very Important
Good question. This shows you are thinking carefully about **error handling**, which is a very good backend habit. 👍

You asked if you should add this after:

```js
const todo = await Todo.create(payload);
```

like this:

```js
if (!todo) throw new AppError("Something went wrong", 500)
```

### Short answer:

❌ **No, you usually should NOT do that after `create()`**

---

## Why?

`Todo.create()` behaves differently from queries like `findOne()`.

### 1️⃣ `create()` either **succeeds or throws an error**

It will:

- ✅ return the created document
    
- ❌ OR throw an error (validation error, DB error, etc.)
    

It **does not return `null`**.

So this situation will basically **never happen**:

```js
const todo = await Todo.create(payload);

if (!todo) {
  // almost impossible
}
```

If something fails, Mongoose will already throw an error like:

- validation error
    
- schema error
    
- database connection error
    

And your `asyncHandler` or `catch` block will handle it.

---

## Where `if (!something)` IS important

You should use it for **queries**, not for `create()`.

Example:

```js
const todo = await Todo.findById(id);

if (!todo) {
  throw new AppError("Todo not found", 404);
}
```

Because `findById()` **can return null**.

---

## Clean Professional Version

Your create controller should look like this:

```js
export const create = asyncHandler(async (req, res) => {

  const payload = req.body;

  const todo = await Todo.create(payload);

  await Activity.create({
    todoId: todo._id,
    action: "create",
    payload
  });

  res.status(201).json({ data: todo });

});
```

No extra `if (!todo)` needed.
