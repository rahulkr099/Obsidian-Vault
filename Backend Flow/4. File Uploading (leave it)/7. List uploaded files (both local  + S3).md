This route is a **simple API that returns information about all uploaded files** (both local and S3).  
It does **not upload anything** — it only **reads data from memory and sends it to the client**.

Let’s understand the **data flow + pseudo logic**.

---

# 1. Big Idea

Your server has a small **in-memory metadata store**:

```text
uploadedFiles[]
```

Every time a file is uploaded, metadata is pushed into this array.

This route simply:

1️⃣ Reads the array  
2️⃣ Counts files  
3️⃣ Sends the data to the client

---

# 2. Pseudo Code

```text
START

Client sends GET request to /files

Server receives request

Read uploadedFiles array

Calculate number of files

Create response object:
    count = number of files
    files = uploadedFiles list

Send response to client

END
```

---

# 13. Important Limitation

Because you are using:

```javascript
uploadedFiles = []
```

All data will be lost if the server restarts.

So in production:

```text
Metadata → Database
Files → S3
```

Example:

```text
MongoDB
PostgreSQL
Redis
```
