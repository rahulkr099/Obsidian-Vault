This code prepares **file upload rules** for your backend using **Multer**.  
Think of it like a **security gate + storage system** for files.

We’ll break it into:

1️⃣ Pseudo code (logic idea)  
2️⃣ Real data flow when a user uploads a file  
3️⃣ Example request → backend → storage

---

# 1. Big Idea

When a user uploads a file, your backend must:

- Check **file type**
    
- Check **file size**
    
- Decide **where to store it**
    
- Generate a **unique filename**
    

Your code sets up **two storage strategies**:

```
Local Disk Storage
Memory Storage (for S3 upload)
```

---

# 2. High-Level Pseudo Code

```
START

Define allowed file types

When user uploads a file:

    Check file MIME type
        IF type not allowed
            reject file
        ELSE
            continue

    Check file size
        IF size > 5MB
            reject file
        ELSE
            continue

    IF storing locally
        save file in upload folder
        generate unique filename

    IF storing in memory (for S3)
        keep file in RAM buffer

END
```

---

# 3. First Layer — Allowed File Types

Your code defines safe file types.

```javascript
const ALLOWED_MIME_TYPES = [
'image/jpeg',
'image/png',
'image/gif',
'application/pdf'
];
```

### Pseudo Code

```
allowedTypes = [jpeg, png, gif, pdf]
```

### Why?

To block dangerous uploads like:

```
.exe
.js
.sh
```

This is **important security practice**.

---

# 4. File Filter (Validation Step)

This function runs **before storing the file**.

```javascript
function fileFilter(req, file, cb)
```

### Pseudo Code

```
function fileFilter(request, file):

    IF file.mimetype not in allowedTypes
        reject upload
    ELSE
        accept upload
```

### Example Upload

User uploads:

```
profile.jpg
```

Multer reads:

```
file.mimetype = image/jpeg
```

Check:

```
image/jpeg ∈ allowedTypes
```

Result:

```
ACCEPT FILE
```

---

### Example Rejection

User uploads:

```
virus.exe
```

MIME type:

```
application/x-msdownload
```

Check:

```
NOT in allowedTypes
```

Result:

```
Error → Unsupported file type
```

Upload stops immediately.

# 5. File Size Limit

```javascript
const FILE_SIZE_LIMIT = 5 * 1024 * 1024;
```

### Calculation

```
5 MB
= 5 × 1024 × 1024
= 5,242,880 bytes
```

### Pseudo Code

```
IF file.size > 5MB
    reject upload
```

Example:

|File|Result|
|---|---|
|image.png (1MB)|Accept|
|resume.pdf (3MB)|Accept|
|video.mp4 (20MB)|Reject|


```
UPLOAD_DIR = /project/uploads
```

# 7. Unique Filename Generator

```javascript
filename: (req, file, cb)
```

Logic:

```
timestamp + random string + original extension
```

### Pseudo Code

```
ext = extension of original file

randomString = generate random hex

filename =
    currentTimestamp + "-" + randomString + ext
```

---

### Practical Example

User uploads:

```
photo.png
```

Process:

```
ext = .png
random = a8f39c2d
timestamp = 1710000000000
```

Generated filename:

```
1710000000000-a8f39c2d.png
```

Saved as:

```
uploads/1710000000000-a8f39c2d.png
```

# 9. Multer Memory Storage

```javascript
multer.memoryStorage()
```

Instead of saving to disk, file is stored **temporarily in RAM**.

### Pseudo Code

```
Store uploaded file in memory buffer
```

Example:

```
file.buffer = <binary data>
```

This is useful for **cloud uploads like AWS S3**.
