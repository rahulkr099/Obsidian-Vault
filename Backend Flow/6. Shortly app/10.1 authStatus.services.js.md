This controller checks whether a user is **still authenticated** by verifying the **access token**.  
It is usually called by the frontend when the page loads to check:

> “Is the user already logged in?”

Let’s understand it using **pseudocode + a real data flow example**.

---

# 1️⃣ Big Picture — Auth Status Flow

Example route:

```javascript
GET /api/v1/auth-status
```

Flow:

```
Frontend (Dashboard / App load)
        │
        ▼
Request /auth-status
        │
        ▼
authStatus Controller
        │
        ▼
Verify JWT accessToken
        │
        ├── Valid → User logged in
        │
        └── Invalid → Ask user to login
```

---

IMPORT jwt library

# 4️⃣ Step 3 — Extract Token

### Your Code

```javascript
const token =
 req?.cookies?.accessToken ||
 req?.body?.accessToken;
```

### Pseudocode

```
TRY to read access token from:

1️⃣ cookies
2️⃣ request body
```

Example values:

```
token = "eyJhbGciOiJIUzI1NiIs..."
```

---

# 5️⃣ Step 4 — Check If Token Exists

### Your Code

```javascript
if (!token)
```

### Pseudocode

```
IF token is missing
    RETURN error response
```

Example response:

```json
{
 "success": false,
 "message": "Token is missing. Please provide a valid token."
}
```

This means the user is **not logged in**.

---

# 6️⃣ Step 5 — Verify Token

### Your Code

```javascript
const decode = jwt.verify(token, process.env.ACCESS_TOKEN_SECRET);
```

### Pseudocode

```
VERIFY access token using secret key
```

Example token payload:

```json
{
 "id": "65fa12abc",
 "email": "rahul@gmail.com",
 "role": "user"
}
```

After verification:

```
decode = {
 id: "65fa12abc",
 email: "rahul@gmail.com"
}
```

---

# 7️⃣ Step 6 — Attach User to Request

### Your Code

```javascript
req.user = decode;
```

### Pseudocode

```
STORE decoded user information in req.user
```

Now other controllers can access:

```
req.user.id
req.user.email
```

---

# 9️⃣ Step 8 — Refresh Cookie Expiry

### Your Code

```javascript
res.cookie("accessToken", accessToken, cookieOptions)
```

### Pseudocode

```
SET accessToken cookie again
UPDATE expiration time
```

Cookie settings:

```
expires → 12 hours
httpOnly → cannot be accessed by JS
secure → only HTTPS in production
sameSite → CSRF protection
```

This is called **sliding session expiration**.

Each request extends the session.

# 🚀 Small Improvement Idea

Currently your function **only returns login status**.

You can also return **user info** so frontend doesn’t need another API call.

Example improved response:

```json
{
 "success": true,
 "message": "User is logged in",
 "user": {
   "id": "65fa12abc",
   "email": "rahul@gmail.com",
   "role": "user"
 }
}
```

This helps frontend quickly load:

```
User dashboard
Profile
Permissions
```
