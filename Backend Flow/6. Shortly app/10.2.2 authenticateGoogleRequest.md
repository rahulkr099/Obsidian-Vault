This controller **refreshes a Google OAuth access token using a Google refresh token**.  
It keeps the user logged in **without asking them to sign in again with Google**.

Think of it like this:

> When the Google access token expires, the refresh token is used to get a new one.

Let’s break it down using **pseudocode + a practical example of how data flows**.

---

# 1️⃣ Big Picture — Google Token Refresh Flow

Example route:

```
POST /api/v1/google/refresh
```

Flow:

```
User Browser
     │
     ▼
Request /google/refresh
     │
     ▼
authenticateGoogleRequest Controller
     │
     ▼
Extract Google Refresh Token
     │
     ▼
Call Google OAuth Server
     │
     ▼
Get New Access Token
     │
     ▼
Set Cookies
     │
     ▼
Return New Tokens
```

# 3️⃣ Step 2 — Extract Refresh Token

### Your Code

```javascript
const incomingRefreshToken =
 req?.cookies?.googleRefreshToken || req?.body?.googleRefreshToken;
```

### Pseudocode

```
TRY to read Google refresh token from:
   cookies
   request body
```

Example value:

```
incomingRefreshToken = "1//0gabcdef12345"
```

---

# 4️⃣ Step 3 — Validate Refresh Token Presence

### Your Code

```javascript
if (!incomingRefreshToken)
```

### Pseudocode

```
IF refresh token is missing
    RETURN unauthorized error
```

Example response:

```json
{
 "success": false,
 "message": "Unauthorized request: Refresh Token is missing"
}
```

This means the user **cannot refresh the session**.

---

# 5️⃣ Step 4 — Request New Tokens from Google

### Your Code

```javascript
const newTokens = await refreshGoogleToken(incomingRefreshToken);
```

### Pseudocode

```
SEND refresh token to Google OAuth server
REQUEST new access token
RECEIVE new tokens
```

Google OAuth endpoint internally used by your helper:

```
https://oauth2.googleapis.com/token
```

Example response from Google:

```json
{
 "access_token": "ya29.a0AfH6SM...",
 "refresh_token": "1//0gabcdef12345",
 "expiry_date": 3600
}
```

Stored in:

```
newTokens
```

# 🔟 Practical Example — Real Data Flow

### Step 1 — User logged in with Google earlier

Browser cookies:

```
googleAccessToken = ya29.a0AfH6SM...
googleRefreshToken = 1//0gabcdef12345
```

---

### Step 2 — Access token expires

Frontend tries request:

```
GET /url/user/status
```

Google API rejects request because token expired.

---

### Step 3 — Frontend calls refresh endpoint

```
POST /api/v1/google/refresh
```

Cookies sent automatically.

---

### Step 4 — Backend extracts refresh token

```
incomingRefreshToken = 1//0gabcdef12345
```

---

### Step 5 — Backend calls Google OAuth

```
POST https://oauth2.googleapis.com/token
```

Google returns new tokens.

---

### Step 6 — Backend sets new cookies

```
googleAccessToken = new_token
googleRefreshToken = same_or_new_token
```

---

### Step 7 — User stays logged in

Frontend can now call APIs again.
