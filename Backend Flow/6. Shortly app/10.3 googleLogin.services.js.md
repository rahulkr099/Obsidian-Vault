This controller handles **Google Login (OAuth 2.0)** for your app.  
It receives an **authorization code from Google**, exchanges it for tokens, fetches the user’s Google profile, creates or finds the user in your database, and finally **creates your own internal authentication token**.

Let’s break it down with **pseudocode + practical data flow**.

---

# 1️⃣ Big Picture — Google Login Flow

When a user clicks **“Login with Google”**, the process looks like this:

```text
User clicks Google Login
        │
        ▼
Google OAuth Screen
        │
        ▼
Google redirects back with authorization code
        │
        ▼
googleLogin Controller
        │
        ▼
Exchange code → Google tokens
        │
        ▼
Fetch user profile from Google
        │
        ▼
Create / find user in database
        │
        ▼
Generate internal middleware token
        │
        ▼
Set cookies and return response
```

---

# 2️⃣ Step-by-Step Pseudocode

## Step 1 — Receive Authorization Code

### Your Code

```javascript
const code = req.query.code;
```

### Pseudocode

```text
READ authorization code from query parameters
IF code does not exist
    RETURN error
```

Example request from Google redirect:

```text
GET /auth/google/callback?code=4/0AVHEtk123abc...
```

So:

```text
code = "4/0AVHEtk123abc..."
```

---

# 3️⃣ Step 2 — Exchange Code for Tokens

### Your Code

```javascript
const googleRes = await oauth2Client.getToken(code);
```

### Pseudocode

```text
SEND authorization code to Google OAuth server
REQUEST access token and refresh token
RECEIVE tokens
```

Behind the scenes Google API:

```text
POST https://oauth2.googleapis.com/token
```

Example response:

```json
{
 "access_token": "ya29.a0AfH6SM...",
 "refresh_token": "1//0gabcdef...",
 "expiry_date": 1710100000000
}
```

Now your server has:

```text
googleAccessToken
googleRefreshToken
```

---

# 4️⃣ Step 3 — Set Tokens in OAuth Client

### Your Code

```javascript
oauth2Client.setCredentials(googleRes.tokens);
```

### Pseudocode

```text
STORE tokens inside OAuth client
ALLOW further API requests to Google
```

---

### Pseudocode

```text
SEND request to Google userinfo API
INCLUDE access token
RECEIVE user profile data
```

Example response:

```json

 "name": "Rahul Kumar",
 "picture": "https://lh3.googleusercontent.com/a/photo.jpg"
}
```

Stored in variables:

```text
email = "rahul@gmail.com"
name = "Rahul Kumar"
picture = "https://..."
```
