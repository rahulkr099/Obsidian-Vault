These two controllers implement the **“Forgot Password → Reset Password”** feature.  
The process works in **two stages**:

1. **Generate a reset token and send email**
    
2. **Verify token and update password**
    

I’ll explain using **pseudocode + practical data flow** so you can clearly see how the data moves.

---

# 1️⃣ Big Picture — Password Reset Flow

```text
User clicks "Forgot Password"
        │
        ▼
POST /reset-password-token
        │
        ▼
Backend generates reset token
        │
        ▼
Token stored in DB (hashed)
        │
        ▼
Email sent to user with reset link
        │
        ▼
User clicks link
        │
        ▼
POST /reset-password
        │
        ▼
Token verified
        │
        ▼
Password updated
```

# 2️⃣ Controller 1 — resetPasswordToken (Send Reset Link)

This function **creates a reset token and sends email to the user**.

---

## Step 1 — Receive Email

### Your Code

```javascript
const email = req.body.email;
```

### Pseudocode

```
READ email from request body
```

Example request:

```http
POST /api/v1/reset-password-token
```

Body:

```json
{
 "email": "rahul@gmail.com"
}
```

---

## Step 2 — Validate Email Format

```javascript
emailRegex.test(email)
```

### Pseudocode

```
IF email format is invalid
    RETURN error
```

Example invalid email:

```
rahulgmail.com
```

Response:

```json
{
 "success": false,
 "message": "Invalid Email Format"
}
```

---

## Step 3 — Check If User Exists

```javascript
const user = await User.findOne({ email });
```

### Pseudocode

```
SEARCH database for user with this email
```

Example database:

```json
{
 "_id": "65fa12abc",
 "email": "rahul@gmail.com",
 "password": "$2b$10$..."
}
```

If user not found:

```json
{
 "success": false,
 "message": "Email is Not Registered"
}
```

---

## Step 4 — Generate Reset Token

```javascript
crypto.randomBytes(20).toString("hex")
```

### Pseudocode

```
GENERATE random token
```

Example:

```
resetPasswordToken = "3fa85f64b12d4c..."
```

---

## Step 5 — Hash Token

```javascript
crypto.createHash("sha256")
```

### Pseudocode

```
HASH the reset token before storing in database
```

Example:

```
originalToken = "3fa85f64b12d4c"
hashedToken = "d0e9f34a6a..."
```

Why?

```
Security → prevents token theft
```

---

## Step 6 — Save Token in Database

```javascript
User.findOneAndUpdate(...)
```

### Pseudocode

```
UPDATE user document with:
    resetPasswordToken
    resetPasswordExpires
```

Example stored record:

```json
{
 "email": "rahul@gmail.com",
 "resetPasswordToken": "d0e9f34a6a...",
 "resetPasswordExpires": 1710000000000
}
```

Expiration time:

```
3 minutes
```

---

## Step 7 — Generate Reset Link

```javascript
const url = `${BASE_URL}/reset-password?token=${resetPasswordToken}&email=${email}`
```

### Pseudocode

```
CREATE password reset link
```

Example link sent to user:

```
https://shortlyapp.in/reset-password?token=3fa85f64b12d4c&email=rahul@gmail.com
```

---

## Step 8 — Send Email

```javascript
mailSender(email, "Password Reset", ...)
```

### Pseudocode

```
SEND email containing reset link
```

Example email:

```
Subject: Password Reset

Click the link to reset your password:
https://shortlyapp.in/reset-password?token=...
```

---

## Step 9 — Send Response

```json
{
 "success": true,
 "message": "Check Your Mail for Resetting the password"
}
```

# 3️⃣ Controller 2 — resetPassword (Update Password)

This controller **verifies the token and updates the password**.

---

## Step 1 — Receive Password Data

```javascript
const { password, confirmPassword, email } = req.body;
```

### Pseudocode

```
READ new password, confirm password, and email
```

Example request:

```http
POST /api/v1/reset-password?token=3fa85f64b12d4c
```

Body:

```json
{
 "email": "rahul@gmail.com",
 "password": "123456",
 "confirmPassword": "123456"
}
```

---

## Step 2 — Validate Password

```
IF password length < 6
    RETURN error
```

Example response:

```json
{
 "success": false,
 "message": "Password must be at least 6 characters"
}
```

---

## Step 3 — Validate Password Match

```
IF confirmPassword != password
    RETURN error
```

Example:

```json
{
 "success": false,
 "message": "Password and Confirm Password Does not Match."
}
```

---

## Step 4 — Extract Token from URL

```javascript
const tokenInUrl = req.query.token;
```

Example URL:

```
/reset-password?token=3fa85f64b12d4c
```

So:

```
tokenInUrl = "3fa85f64b12d4c"
```

---

## Step 5 — Hash Token

```javascript
hashedToken = sha256(tokenInUrl)
```

### Pseudocode

```
HASH the token from URL
```

Example:

```
tokenInUrl = 3fa85f64b12d4c
hashedToken = d0e9f34a6a...
```

---

## Step 6 — Find User With Token

```javascript
User.findOne({ email, resetPasswordToken: hashedToken })
```

### Pseudocode

```
SEARCH database for matching email and token
```

Example record:

```json
{
 "email": "rahul@gmail.com",
 "resetPasswordToken": "d0e9f34a6a...",
 "resetPasswordExpires": 1710000000000
}
```

---

## Step 7 — Check Token Expiration

```
IF resetPasswordExpires < currentTime
    RETURN expired error
```

Example response:

```json
{
 "success": false,
 "message": "Token is Expired, Please Regenerate Your Token."
}
```

---

## Step 8 — Hash New Password

```javascript
bcrypt.hash(password, 10)
```

### Pseudocode

```
HASH new password
```

Example:

```
123456 → $2b$10$sdjfhskdjf...
```

---

## Step 9 — Update Password

```javascript
User.findOneAndUpdate(...)
```

### Pseudocode

```
UPDATE user document:
    password
REMOVE reset token
REMOVE expiration
```

Updated record:

```json
{
 "email": "rahul@gmail.com",
 "password": "$2b$10$sdjfhskdjf...",
 "resetPasswordToken": null
}
```

---

## Step 10 — Send Success Response

```json
{
 "success": true,
 "message": "Password Reset Successful"
}
```

# 🚀 One Security Improvement

Your system is already strong, but production systems also add:

```
Rate limiting for reset requests
```

Example:

```
Max 3 reset emails per hour
```

This prevents:

```
Email spam
Brute force attacks
```
