These files are **utility services** used across your backend.  
They provide reusable functions for:

- 🔐 JWT token generation
    
- 📧 Sending emails
    
- 🔑 Google OAuth authentication
    
- 🔄 Refreshing Google tokens
    

Instead of repeating logic in controllers, these **utilities centralize the functionality**.

Let’s break them down with **pseudocode + practical data flow**.

---

# 1️⃣ JWT Token Generator Utilities

File contains:

```text
generateAccessToken
generateRefreshToken
generateGoogleMiddlewareToken
```

These functions create **JWT tokens for authentication**.

---

## Code

```javascript
export const generateAccessToken = (payload) => {
    return jwt.sign(payload, process.env.ACCESS_TOKEN_SECRET,{
        expiresIn:'39m'
    })
}
```

---

## Pseudocode

```text
FUNCTION generateAccessToken(payload)

    USE jwt.sign()

    INPUT:
        payload → user data

    SECRET:
        ACCESS_TOKEN_SECRET

    EXPIRATION:
        39 minutes

    RETURN signed JWT token
```

---

## Example

Payload:

```json
{
 "id":"65fa12abc",
 "email":"rahul@gmail.com",
 "role":"user"
}
```

Generated token:

```text
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

This token is used for:

```text
Protected API routes
```

Example request:

```http
GET /url/user/status
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

---

## Refresh Token Generator

```javascript
export const generateRefreshToken = (payload)
```

### Pseudocode

```text
FUNCTION generateRefreshToken(payload)

    SIGN JWT with REFRESH_TOKEN_SEC
    SET expiration = 40 minutes
    RETURN refresh token
```

Purpose:

```text
Generate new access tokens when access token expires
```

---

## Google Middleware Token

```javascript
generateGoogleMiddlewareToken()
```

### Pseudocode

```text
CREATE internal JWT for Google authenticated users
```

Why needed?

```text
Google verifies identity
Your backend manages sessions
```

So you convert:

```text
Google Access Token → Backend JWT
```

# 2️⃣ Email Sender Utility

This file sends **emails using Nodemailer**.

---

## Code

```javascript
export const mailSender = async (email,title,body,text)
```

---

## Pseudocode

```text
FUNCTION mailSender(email,title,body)

CREATE nodemailer transporter
    HOST → MAIL_HOST
    PORT → 465
    SECURE → true

AUTHENTICATE using
    MAIL_USER
    MAIL_PASS

SEND email with:
    from
    to
    subject
    html body

RETURN response
```

---

## Example

When user requests password reset:

Controller calls:

```javascript
mailSender(
 "rahul@gmail.com",
 "Password Reset",
 "<a href='reset-link'>Reset Password</a>"
)
```

Email sent:

```text
To: rahul@gmail.com
Subject: Password Reset

Click the link below to reset your password:
https://shortlyapp.in/reset-password?token=abc123
```

# 3️⃣ Google OAuth Client Utility

This initializes Google OAuth.

---

## Code

```javascript
export const oauth2Client = new google.auth.OAuth2(
    GOOGLE_CLIENT_ID,
    GOOGLE_CLIENT_SECRET,
    'postmessage'
)
```

---

## Pseudocode

```text
CREATE OAuth2 client

CONFIGURE with:
    client_id
    client_secret
    redirect_uri
```

This client is used to:

```text
Exchange authorization code
Refresh access tokens
Call Google APIs
```

Example:

```javascript
oauth2Client.getToken(code)
```

# 4️⃣ Google Token Refresh Utility

Function:

```javascript
refreshGoogleToken(refreshToken)
```

---

## Pseudocode

```text
FUNCTION refreshGoogleToken(refreshToken)

SET OAuth client credentials
    refresh_token = refreshToken

CALL Google API to refresh token

RECEIVE new credentials
    access_token
    expiry_date

RETURN credentials
```

---

## Example Data Flow

User logged in with Google.

Cookies stored:

```text
googleAccessToken = ya29.a0AfH6SM...
googleRefreshToken = 1//0gabcdef...
```

Later:

```text
Access token expires
```

Frontend calls:

```http
POST /google/refresh
```

Backend executes:

```javascript
refreshGoogleToken("1//0gabcdef...")
```

Google returns:

```json
{
 "access_token":"ya29.NEW_TOKEN...",
 "expiry_date":3600
}
```

New token stored in cookies.

User stays logged in.

# 5️⃣ Practical End-to-End Data Flow

Let’s see **how these utilities work together**.

---

### 1️⃣ User logs in

Controller calls:

```javascript
generateAccessToken(user)
generateRefreshToken(user)
```

Response:

```text
accessToken cookie
refreshToken cookie
```

---

### 2️⃣ User forgets password

Controller calls:

```javascript
mailSender()
```

Email sent.

---

### 3️⃣ User logs in with Google

Controller uses:

```javascript
oauth2Client.getToken()
```

---

### 4️⃣ Google token expires

Controller calls:

```javascript
refreshGoogleToken()
```

New token generated.
