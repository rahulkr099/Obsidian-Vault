This middleware protects your routes.  
It **checks if a user is logged in by verifying a JWT token** before allowing access.

Let’s break it into **simple pseudocode + real request examples** so you clearly see how data flows.

---

# 1️⃣ Purpose of This Middleware

Your middleware runs **before protected routes**.

Example protected route:

```javascript
router.post('/shorten', auth, handleShortenURL);
```

Flow:

```
Client Request
      │
      ▼
Auth Middleware
      │
      ├── Token Valid → Continue to Controller
      │
      └── Token Invalid → Return 401 Error
```

So **no token = no access**.

# 4️⃣ Step 3 — Extract Token

### Your code

```javascript
const token =
 req?.cookies?.accessToken ||
 req?.body?.accessToken ||
 req?.headers?.["authorization"]?.replace("Bearer ", "");
```

### Pseudocode

```
TRY to get token from:

1. cookies
2. request body
3. authorization header
```

---

## Practical Example 1 — Token from Cookie

Request:

```
POST /url/shorten
```

Header:

```
Cookie: accessToken=eyJhbGciOiJIUzI1NiIs...
```

Result:

```
req.cookies.accessToken → token found
```

## Practical Example 2 — Token from Header

Request:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

Code extracts:

```
Bearer eyJhbGci → remove "Bearer "
```

Final:

```
token = eyJhbGciOiJIUzI1NiIs...
```

### Pseudocode

IF normal token not found
    CHECK google login token in cookies

# 6️⃣ Step 5 — If Both Tokens Missing

### Your code

```javascript
if (!googleToken && !token)
```

### Pseudocode

```
IF no token AND no google token
    RETURN 401 Unauthorized
```

Response

```json
{
 "success": false,
 "message": "Token is missing. Please provide a valid token."
}
```

Flow stops here.

---

# 7️⃣ Step 6 — Verify Token

### Your code

```javascript
decode = jwt.verify(token, process.env.ACCESS_TOKEN_SECRET);
```

### Pseudocode

```
VERIFY token using secret key
IF verification successful
    RETURN decoded user data
ELSE
    THROW error
```

Example token payload

```
{
 userId:"123",
 email:"rahul@gmail.com",
 role:"user"
}
```

After verification:

```
decode = {
 userId:"123",
 email:"rahul@gmail.com"
}
```

---

# 8️⃣ Step 7 — Attach User to Request

### Your code

```javascript
req.user = decode;
```

### Pseudocode

```
STORE decoded user data inside req.user
```

Now all controllers can access the logged-in user.

Example

```
req.user = {
 userId: "123",
 email: "rahul@gmail.com"
}
```

# 🚀 One Improvement Idea (Production Level)

Your token extraction is good, but you can simplify it.

Cleaner version:

```
1 Try accessToken
2 Try Authorization header
3 Try Google token
```

Also many production apps add:

```
refresh token validation
```

Flow becomes:

```
Access Token expired
      │
      ▼
Use Refresh Token
      │
      ▼
Generate new Access Token
```

This improves **user experience + security**.
