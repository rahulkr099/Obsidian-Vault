This function is your **signup controller**.  
It receives user data from the frontend, checks it, secures the password, stores the user in the database, and sends a response back.

Let’s understand it using **clear pseudocode + a real example of data flow**.

---

# 1️⃣ Overall Request Flow

Before this controller runs, the request already passed through **router and middlewares**.

Typical flow:

```
Client (Frontend Form)
        │
        ▼
POST /api/v1/signup
        │
        ▼
Router
        │
        ▼
Signup Controller
        │
        ▼
Database (MongoDB)
        │
        ▼
Response to Client
```

# 6️⃣ Hash the Password

### Your Code

```javascript
hashedPassword = await bcrypt.hash(password, 10);
```

### Pseudocode

```
HASH password using bcrypt
```

Example:

Original password:

```
123456
```

Hashed password:

```
$2b$10$9g1nQ2T...
```

Why hashing?

```
Security
```

Even if database is leaked, passwords remain protected.
