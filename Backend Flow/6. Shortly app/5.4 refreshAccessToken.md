This controller handles **refreshing an expired access token** using a **refresh token**.  
It keeps the user logged in without asking them to login again.

Let’s understand **how data flows** using **pseudocode + a real example**.

---

# 1️⃣ Big Picture — Token Refresh Flow

When the **access token expires**, the frontend calls `/refresh`.

```text
Client (Browser)
      │
      ▼
POST /api/v1/refresh-token
      │
      ▼
Refresh Token Controller
      │
      ▼
Verify Refresh Token
      │
      ▼
Find User in Database
      │
      ▼
Generate New Tokens
      │
      ▼
Update Database
      │
      ▼
Send New Tokens + Cookies
```

This allows **silent authentication**.

# 🚀 One Advanced Improvement Idea

Production systems also implement **refresh token rotation detection**.

Example attack:

```text
Hacker steals refresh token
```

Solution:

```text
Store previous refresh tokens
Detect reuse
Invalidate all sessions
```

Flow becomes:

```text
Login
  │
  ▼
Refresh Token Rotation
  │
  ▼
Detect token reuse
  │
  ▼
Force logout all sessions
```

This is used by **Google, Facebook, and large SaaS systems**.
