This code is the **query builder layer** of your Ecommerce Product API.  
Its job is to convert **URL query parameters → MongoDB query objects**.

So the backend can support:

- Filtering products
    
- Sorting products
    
- Pagination (handled later usually)
    

Think of it as a **translator between client request and database query**.

---

# 1. Big Idea

Client sends request like:

```http
GET /products?category=phone&minPrice=20000&sort=price,-rating
```

Your backend converts this into:

```javascript
{
 category: "phone",
 price: { $gte: 20000 }
}
```

And sorting:

```javascript
{ price: 1, rating: -1 }
```

Then MongoDB runs:

```javascript
Product.find(filter).sort(sort)
```

# 3. Pseudo Code — Filter Builder

```text
START

Create empty filter object

IF category exists
    filter.category = category

IF brand exists
    filter.brand = brand

IF tags exist
    split tags by comma
    filter.tags = match any tag

IF inStock = true
    filter.stock > 0

IF minPrice or maxPrice exists
    create price filter
    price >= minPrice
    price <= maxPrice

IF minRating exists
    rating >= minRating

Return filter object

END
```

# 4. Practical Example of Filter Flow

### Client Request

```http
GET /products?category=phone&brand=samsung&minPrice=20000&maxPrice=60000
```

---

### Step 1 — Query Params

Express reads:

```javascript
req.query = {
 category: "phone",
 brand: "samsung",
 minPrice: "20000",
 maxPrice: "60000"
}
```

---

### Step 2 — buildFilter(req.query)

Function starts with:

```javascript
const f = {}
```

---

### Step 3 — Category Filter

```javascript
f.category = "phone"
```

Filter becomes:

```javascript
{
 category: "phone"
}
```

---

### Step 4 — Brand Filter

```javascript
f.brand = "samsung"
```

Filter becomes:

```javascript
{
 category: "phone",
 brand: "samsung"
}
```

---

### Step 5 — Price Range

```javascript
f.price = {
 $gte: 20000,
 $lte: 60000
}
```

Final filter object:

```javascript
{
 category: "phone",
 brand: "samsung",
 price: {
  $gte: 20000,
  $lte: 60000
 }
}
```

---

# 5. MongoDB Query Generated

Your API later runs something like:

```javascript
Product.find({
 category: "phone",
 brand: "samsung",
 price: { $gte: 20000, $lte: 60000 }
})
```

MongoDB returns matching products.

# 6. Tags Filter Example

### Request


```javascript

```

MongoDB query:

```javascript
Product.find({
 tags: { $in: ["phone", "samsung"] }
})
```

Meaning:

```text
Products containing ANY of these tags
```

# 9. Pseudo Code — Sort Parser

```text
START

IF no sort parameter
    return default sort by createdAt descending

Split sort string by commas

FOR each field
    IF field starts with "-"
        direction = descending
        remove "-"
    ELSE
        direction = ascending

    add field to sort object

Return sort object

END
```

# 10. Sorting Example

### Client Request

```http
GET /products?sort=price,-rating
```

---

### Step 1 — Parse Sort

Input string:

```text
price,-rating
```

Split:

```text
["price", "-rating"]
```

---

### Step 2 — Build Sort Object

Result:

```javascript
{
 price: 1,
 rating: -1
}
```

Meaning:

```text
Sort by price ascending
Then rating descending
```

# 11. MongoDB Sort Query

```javascript
Product.find(filter).sort({

})
```

# 15. Small Pro-Level Improvement Idea

A powerful improvement is **field validation for sort**.

Example:

```javascript
allowedSortFields = ["price","rating","createdAt"]
```

So malicious queries like:

```http
?sort=password
```

cannot break your API.
