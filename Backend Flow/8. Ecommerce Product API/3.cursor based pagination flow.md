This block is the **cursor-based pagination engine** of your Product API.  
It is designed for **high-performance product feeds**, similar to how **Instagram, Twitter, and Amazon infinite scrolling** work.

Instead of saying **“give me page 2”**, the client says:

> “Give me products **after this last item**.”

That last item reference is the **cursor**.

---

# 1. Big Idea

Cursor pagination works like this:

1️⃣ Client requests products  
2️⃣ Server returns products + **nextCursor**  
3️⃣ Client sends next request using that cursor  
4️⃣ Server returns the **next batch**

This avoids slow database operations like large **skip()** queries.

```text
      ▼

      ▼

      │
      ▼

      │
      ▼

      │
      ▼

      ▼

      │
      ▼

      │
      ▼
Compute facets
      │
      ▼
Send response
```

---

# 3. Pseudo Code

```text
START

IF cursor exists

    Decode cursor from base64
    Convert cursor value into createdAt date

    Update filter:
        createdAt < cursorDate

    Query database
        sort by createdAt descending
        limit results to limit + 1

    IF results > limit
        hasMore = true
        remove extra result
    ELSE
        hasMore = false

    IF hasMore
        generate nextCursor using last document createdAt

    Add response headers
        X-Result-Count
        X-Next-Cursor

    Generate ETag hash for caching

    IF facet enabled
        run aggregation for analytics

    Return response

END
```

# 6. Cursor Decoding

Your code:

```javascript
Buffer.from(cursor, 'base64').toString()
```

Result:

```text
2026-03-07T10:00:00.000Z
```

Pseudo logic:

```text
cursorDate = decoded cursor
```

---

# 7. Update MongoDB Filter

Your code:

```javascript
filter.createdAt = { $lt: new Date(decoded) }
```

Filter becomes:

```javascript
{
 createdAt: { $lt: "2026-03-07" }
}
```

Meaning:

```text
Only fetch older products
```

# 8. Query Database

Your code:

```javascript
Product.find(filter)
```

# 9. Detect More Results

Your code:

```javascript
const hasMore = docs.length > limitNum
```

Example:

```text
4 > 3 = true
```

So:

```text
There is another page
```

```javascript

```

# 10. Generate Next Cursor

Your code:

```javascript
Buffer.from(lastItem.createdAt).toString('base64')
```

Example:

```text
lastItem.createdAt = 2026-03-04
```

Encoded cursor:

```text
MjAyNi0wMy0wNFQxMDowMDowMC4wMDBa
```

Client will use this for the **next request**.

# 12. ETag Generation (Caching)

Your code:

```javascript
crypto.createHash('md5')
```

Pseudo logic:

```text
Generate hash of response data
```

Example:

```text
etag = "8d9f7c2e5b..."
```

Purpose:

```text
Client can cache response
```

If client sends:

```http
If-None-Match: 8d9f7c2e5b
```

Server can reply:

```text
304 Not Modified
```

Saving bandwidth.

# 15. Why Cursor Pagination Is Powerful

Compared to **page-based pagination**:

|Page Pagination|Cursor Pagination|
|---|---|
|Uses skip()|Uses index|
|Slow for large data|Very fast|
|Bad for real-time feeds|Perfect for feeds|

That’s why companies use it:

```text
Twitter timeline
Instagram feed
YouTube comments
Amazon infinite scroll
```
