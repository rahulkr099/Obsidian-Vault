This part of the code implements **page-based pagination** (traditional pagination).  
It is used when **cursor pagination is not provided**. This is common in **admin dashboards, product lists, and search results**.

Here the client asks:

> “Give me **page 2 with 20 products per page**.”

Let’s break down the **logic, pseudo code, and real data flow**.

---

# 1. Big Idea

Page pagination works using two numbers:

```
page
limit
```

Example request:

```http
GET /products?page=2&limit=20
```

Backend calculates:

```
skip = (page - 1) × limit
```

Then MongoDB query becomes:

```
skip first N documents
return next limit documents
```

# 4. Step-by-Step Practical Example

### Request

```http
GET /products?page=2&limit=5
```

Suppose database contains **23 products**.

---

# 5. Step 1 — Read Query Params

Express reads:

```javascript
req.query = {
 page: "2",
 limit: "5"
}
```

Your code converts them:

```javascript
pageNum = 2
limitNum = 5
```

---

# 6. Step 2 — Calculate Skip

Your code:

```javascript
skip = (pageNum - 1) * limitNum
```

Calculation:

```
skip = (2 - 1) × 5
skip = 5
```

Meaning:

```
Skip first 5 products
```

---

# 7. Step 3 — Build MongoDB Query

Your code:

```javascript
Product.find(filter)
.skip(skip)
.limit(limitNum)
```

Query becomes:

```javascript
Product.find(filter)
.skip(5)
.limit(5)
```

So MongoDB returns products:

```
Product 6
Product 7
Product 8
Product 9
Product 10
```

---

# 8. Step 4 — Sorting

If **text search is used**:

```javascript
query.sort({ score: { $meta: "textScore" }, ...sortObj })
```

Example request:

```
GET /products?q=samsung
```

MongoDB sorts results by **relevance score**.

Example order:

```
Samsung Galaxy S23
Samsung Charger
Samsung Galaxy A54
```

If no search query:

```
sort by provided fields
```

Example:

```
GET /products?sort=price,-rating
```

Sorting:

```
price ascending
rating descending
```

---

# 9. Step 5 — Execute Queries in Parallel

Your code uses:

```javascript
Promise.all()
```

This runs two database operations **at the same time**.

```javascript
[docs, total]
```

Query 1:

```
fetch products
```

Query 2:

```
count total documents
```

Example results:

```
docs = 5 products
total = 23
```

---

# 10. Step 6 — Calculate Total Pages

Your code:

```javascript
totalPages = Math.ceil(total / limitNum)
```

Example:

```
total = 23
limit = 5
```

Calculation:

```
23 / 5 = 4.6
ceil = 5
```

So:

```
totalPages = 5
```

---

# 11. Step 7 — Generate Pagination Links

Your code builds **navigation links**.

Example:

```
GET /products?page=2
```

Links created:

```
prev → page 1
next → page 3
```

Header sent:

```
Link:
<http://localhost/products?page=1>; rel="prev",
<http://localhost/products?page=3>; rel="next"
```

This follows **HTTP RFC pagination standards**.
