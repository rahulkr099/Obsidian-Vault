# 🚀 1. What is HTTP?

HTTP (**HyperText Transfer Protocol**) is the communication protocol between **client (browser/app)** and **server (backend)**.

👉 It works on a **request → response model**

---

# 🧠 2. Stateless Nature

- HTTP is **stateless**
    
- Server **does NOT remember previous requests**
    

### Example:

```
Request 1 → Login
Request 2 → Get profile
```

Server doesn’t automatically know you are logged in.

👉 That’s why we use:

- Cookies
    
- Sessions
    
- JWT tokens
    

💡 Interview Tip:

> “HTTP is stateless, so we maintain state using tokens or sessions.”

---

# 🏗️ 3. Client-Server Model

- **Client** → sends request (browser, mobile app)
    
- **Server** → processes and responds
    

### Flow:

```
Client → Request → Server → Response → Client
```

---

# 📈 4. Evolution of HTTP

- **HTTP/1.0**
    
    - One request per connection
        
    - Slow
        
- **HTTP/1.1**
    
    - Persistent connections (keep-alive)
        
    - Pipelining
        
- **HTTP/2**
    
    - Multiplexing (multiple requests in one connection)
        
    - Faster
        
- **HTTP/3**
    
    - Uses QUIC (UDP based)
        
    - Even faster, better performance
        

---

# 📦 5. HTTP Messages

## Request Structure:

```
GET /users HTTP/1.1
Host: example.com
Authorization: Bearer token

Body (optional)
```

## Response Structure:

```
HTTP/1.1 200 OK
Content-Type: application/json

{ "name": "Rahul" }
```

---

# 🎯 6. Why do we need HTTP Headers?

Headers carry **metadata (extra info)**

### Example:

- Authentication → `Authorization`
    
- Content type → `Content-Type`
    
- Caching → `Cache-Control`
    

💡 Without headers:

- Server won’t know format
    
- No auth
    
- No caching control
    

---

# 📚 7. Types of Headers

### 1. Request Headers

- Sent by client
    
- Example: `Authorization`, `User-Agent`
    

### 2. Response Headers

- Sent by server
    
- Example: `Content-Type`, `Set-Cookie`
    

### 3. General Headers

- Apply to both
    
- Example: `Cache-Control`
    

### 4. Entity Headers (Representation headers)

- Describe body
    
- Example: `Content-Length`, `Content-Type`
    

---

# 🔧 8. HTTP Methods

|Method|Use|
|---|---|
|GET|Fetch data|
|POST|Create|
|PUT|Update (full)|
|PATCH|Update (partial)|
|DELETE|Remove|
|OPTIONS|Check allowed methods|

💡 Interview Tip:

> “Use GET for fetching, POST for creating, PUT for replacing, PATCH for partial updates.”

---

# ⚖️ 9. Idempotent vs Non-Idempotent

### Idempotent:

Same request → same result

Examples:

- GET
    
- PUT
    
- DELETE
    

👉 Calling 10 times = same result

---

### Non-Idempotent:

Each request changes state

Examples:

- POST
    

👉 Calling multiple times creates multiple records

---

# 🌐 10. OPTIONS Method & CORS

## What is CORS?

Cross-Origin Resource Sharing

👉 When frontend and backend are on different domains

---

## CORS Flow:

1. Browser sends **preflight request**
    

```
OPTIONS /api
Origin: http://frontend.com
```

2. Server responds:
    

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST
```

3. Then actual request is sent
    

---

💡 Important Headers:

- `Access-Control-Allow-Origin`
    
- `Access-Control-Allow-Methods`
    
- `Access-Control-Allow-Headers`
    

---

- Streaming
    
- Large responses

# 🔒 18. SSL, TLS, HTTPS

## HTTP vs HTTPS

- HTTP → Not secure
    
- HTTPS → Secure
    

---

## TLS (Transport Layer Security)

Encrypts data

---

### How HTTPS Works:

1. Client sends request
    
2. Server sends certificate
    
3. Secure key exchange
    
4. Encrypted communication
    

---

💡 Interview Tip:

> “HTTPS uses TLS to encrypt communication and prevent MITM attacks.”

# 🎯 SECTION 1: Core HTTP Understanding

### ❓ Q1: What is HTTP and how does it work?

👉 Answer:  
HTTP is a **stateless protocol** used for communication between client and server. It follows a **request-response model** where the client sends a request and the server responds with data.

---

### ❓ Q2: Why is HTTP called stateless?

👉 Answer:  
Because each request is independent and the server does not remember previous requests. We use **cookies, sessions, or JWT** to maintain state.

---

### ❓ Q3: What is the structure of an HTTP request?

👉 Answer:

- Request line (method + URL)
    
- Headers
    
- Body (optional)
    

---

### ❓ Q4: Difference between HTTP and HTTPS?

👉 Answer:

- HTTP → not secure
    
- HTTPS → encrypted using TLS
    
- Prevents MITM attacks
    

---

# 🧠 SECTION 2: Headers & Methods

### ❓ Q5: Why do we need headers?

👉 Answer:  
Headers carry metadata like:

- Content type
    
- Authorization
    
- Caching info
    

---

### ❓ Q6: What is Content-Type header?

👉 Answer:  
It tells the server what type of data is being sent (e.g., JSON, form-data).

---

### ❓ Q7: Difference between PUT and PATCH?

👉 Answer:

- PUT → full update (replace entire resource)
    
- PATCH → partial update
    

---

### ❓ Q8: Difference between POST and PUT?

👉 Answer:

- POST → create new resource (non-idempotent)
    
- PUT → update existing (idempotent)
    

---

# ⚖️ SECTION 3: Idempotency (Very Important)

### ❓ Q9: What is idempotency?

👉 Answer:  
If making the same request multiple times results in the same outcome.

---

### ❓ Q10: Which HTTP methods are idempotent?

👉 Answer:

- GET
    
- PUT
    
- DELETE
    

---

### ❓ Q11: Why POST is not idempotent?

👉 Answer:  
Because each request creates a new resource.

---

# 🌐 SECTION 4: CORS (Most Asked 🔥)

### ❓ Q12: What is CORS?

👉 Answer:  
CORS allows or restricts requests from different origins (domains).

---

### ❓ Q13: What is a preflight request?

👉 Answer:  
A browser sends an **OPTIONS request** before actual request to check permissions.

---

### ❓ Q14: Important CORS headers?

👉 Answer:

- Access-Control-Allow-Origin
    
- Access-Control-Allow-Methods
    
- Access-Control-Allow-Headers
    

---

### ❓ Q15: What happens if CORS is not configured?

👉 Answer:  
Browser blocks the request (even if backend works fine).

---

# 📊 SECTION 5: Status Codes

### ❓ Q16: Difference between 401 and 403?

👉 Answer:

- 401 → Not authenticated
    
- 403 → Authenticated but not allowed
    

---

### ❓ Q17: When to use 201?

👉 Answer:  
When a new resource is created.

---

### ❓ Q18: Difference between 200 and 204?

👉 Answer:

- 200 → success with response body
    
- 204 → success with no content
    

---

# ⚡ SECTION 6: Performance & Optimization

### ❓ Q19: What is HTTP caching?

👉 Answer:  
Storing response so it can be reused instead of calling server again.

---

### ❓ Q20: What is ETag?

👉 Answer:  
A unique identifier for a resource used to check if it changed.

---

### ❓ Q21: What is compression?

👉 Answer:  
Reducing response size using gzip or brotli.

---

### ❓ Q22: What is keep-alive?

👉 Answer:  
Reusing same TCP connection for multiple requests.

---
