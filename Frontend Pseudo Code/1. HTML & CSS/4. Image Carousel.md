# **Pseudocode: Image Carousel**

---

## **START**

---

## **INITIALIZATION**

```
SET index = 0
GET all images inside the carousel and store them in a list called images
```

---

## **PROCEDURE showImage(i)**

```
FOR each image in images
    REMOVE "active" class from image
END FOR

ADD "active" class to images[i]
```

---

## **PROCEDURE next()**

```
INCREASE index by 1

IF index is equal to number of images
    SET index to 0
END IF

CALL showImage(index)
```

---

## **PROCEDURE prev()**

```
DECREASE index by 1

IF index is less than 0
    SET index to last image index
END IF

CALL showImage(index)
```

---

## **USER INTERACTION**

```
WHEN user clicks "Next" button
    CALL next()

WHEN user clicks "Prev" button
    CALL prev()
```

---

## **END**

---

## üß† Simple Explanation (In Plain Words)

- Only **one image is visible** at a time
    
- The `"active"` class controls visibility
    
- `Next` moves forward
    
- `Prev` moves backward
    
- Index resets automatically to stay within range
    

This wrapping logic makes the carousel **loop endlessly**.
